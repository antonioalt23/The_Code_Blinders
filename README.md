## **3. Instrucción para la instalar la distribucion Anaconda de Python**

## **4.Sugerencias de editor de texto**
En nuestro flujo de trabajo de desarrollo de Python recomendado, escribirá scripts y módulos de Python (archivos * .py) en un editor de texto. Luego, ejecutará esos scripts desde su terminal. Querrá un editor de texto capaz para desarrollar su código. Existen muchos editores de texto capaces, pero recomendamos tres.<br>
1. [Atom](https://atom.io/)
1. [Sublime Text 3](https://www.sublimetext.com/)
1. [Vim](https://www.vim.org/)

Atom y Vim son completamente gratuitos. Una versión de prueba de Sublime Text 3 está disponible de forma gratuita, pero una versión con licencia cuesta $ 70 (dólares estadounidenses). En las siguientes subsecciones, damos algunos de los detalles de cada uno de los tres editores de texto anteriores.

## **4.1. Atom**
 [Atom](https://atom.io/) es un editor de texto de código abierto desarrollado por personas de GitHub.com. Este editor tiene todas las características de Sublime Text 3, pero también permite a los usuarios una personalización completa. Además, ha pasado un tiempo desde que los usuarios de Atom han superado la masa crítica necesaria para que el editor siga avanzando con las adiciones más innovadoras.

Hay varios paquetes que querrá instalar con Atom. Una vez que Atom está instalado, puede agregar paquetes navegando Atom-> Preferencias-> Instalar y luego escribiendo el nombre del paquete que le gustaría instalar.<br>

Para trabajar con Python, recomendamos instalar los siguientes paquetes:<br>

  *   MagicPython
  *   python-indent
  *   tabs-to-spaces
  *   minimap
  *   open-recent
  *   linter-python-pep8

  
Para el desarrollo con GitHub recomendamos:<br>

  *   merge-conflict


Si usa LaTex en este editor, los siguientes paquetes son útiles: 

  *   atom-latex
  *   latextools
  *   autocomplete-bibtex
  *   dictionary
  *   latexer
  *   pdf-view

Además, también querrá descargar el visor de PDF [Skim](https://skim-app.sourceforge.io/) para ayudar a mostrar archivos PDF compilados de TeX con Atom. 

## **4.2. Sublime Text 3**

Sublime Text 3 es el editor de texto de software privado más utilizado y versátil. Tiene una tremenda flexibilidad, así como el pulido de un software profesional. Sublime Text 3 costará $ 70 por una licencia, aunque puede usar una versión de prueba indefinidamente sin cargo mientras solo tiene que sufrir recordatorios frecuentes para comprar la versión completa.


## **4.3. Vim**

Vim es gratis y muy poderoso. Vim es el editor de texto preferido por los desarrolladores de núcleo duro. La curva de aprendizaje para usar vim es un poco más empinada que la de Atom y Sublime Text 3, pero también tiene algunas ventajas para una programación eficiente. Vim tiene una navegación que no usa un mouse o trackpad. Eventualmente, tus dedos nunca abandonan tu teclado. Además, la mayoría de los terminales tienen Vim integrado, por lo que puede usar Vim para editar scripts y módulos sobre la marcha con su sesión de terminal.

## **7. Jupyter Notebooks**
[Jupyter notebooks](https://jupyter.org/) son archivos que terminan con el sufijo * .ipynb. Estos cuadernos se abren en un entorno de navegador y son una aplicación web de código abierto que combina texto instructivo con código ejecutable y modificable en vivo para muchas plataformas de programación diferentes (por ejemplo, Python, R, Julia). Los cuadernos de Jupyter son una herramienta ideal para enseñar programación, ya que proporcionan el código para que un usuario lo ejecute y también proporcionan el contexto y la explicación del código. Hemos proporcionado varios cuadernos de Jupyter en la carpeta [Tutorials](https://github.com/OpenSourceEcon/BootCamp2017/tree/master/Tutorials) de este repositorio.

Estos cuadernos solían ser específicos de Python y, por lo tanto, se llamaban cuadernos iPython (de ahí el sufijo * .ipynb). Pero los cuadernos de Jupyter ahora soportan muchos lenguajes de programación, aunque el nombre todavía rinde homenaje a Python con el vestigio "py" en "Jupyter". Los cuadernos ejecutan código desde el núcleo del lenguaje de programación específico en su máquina local.

La capacidad de los cuadernos de Jupyter se instalará automáticamente con su descarga de la distribucion Anaconda de Python. Si no descargó la distribución Anaconda de Python, puede descargar los cuadernos de Jupyter por separado siguiendo las instrucciones en la página de [install page](https://jupyter.org/install.html).

## **7.1. Iniciando un Jupyter Notebook**
Una vez instalado Jupyter, ya sea a través de Anaconda o del sitio web de Jupyter, puede abrir un cuaderno de Jupyter siguiendo los siguientes pasos.<br>

1. Navegue en su terminal a la carpeta en la que residen los archivos del cuaderno de Jupyter. En el caso de los tutoriales del cuaderno de Jupyter en este repositorio, debería navegar al directorio ~/OG-JRC/Tutorials/.

2. Escriba *jupyter notebook* en el indicador de la terminal.

3. Se abrirá una sesión de cuaderno de Jupyter en su navegador, mostrando los archivos * .ipynb disponibles en ese directorio.

4. Haga doble clic en el cuaderno de Jupyter que le gustaría abrir.

## **7.2. Usando Jupyter Notebook**
Una vez que haya abierto un cuaderno de Jupyter, encontrará que el cuaderno tiene dos tipos principales de celdas: celdas de Markdown y celdas de código. Las celdas de rebajas han formateado el texto de rebajas del cuaderno de Jupyter y sirven principalmente para presentar el contexto de las celdas de codificación. Una referencia para las opciones de rebajas en los cuadernos de Jupyter se encuentra en la [Jupyter markdown documentation page](https://jupyter-notebook.readthedocs.io/en/latest/examples/Notebook/Working%20With%20Markdown%20Cells.html)<br>

Puede editar una celda de Markdown en un cuaderno de Jupyter haciendo doble clic en la celda y luego haciendo los cambios. Asegúrese de que el cuadro de tipo de celda en el medio de la barra de menú superior esté configurado en Markdown. Para implementar sus cambios en la celda de Markdown, escriba `Shift-Enter`.<br>

Una celda de Código tendrá un In [ ]: inmediatamente a la izquierda de la celda para la entrada. El código en esa celda se puede ejecutar escribiendo ` `Shift-Enter`. Para una celda de Código, el cuadro de tipo de celda en el medio de la barra de menú superior dice Código.


## **7.3. Cerrando Jupyter notebook**
Cuando haya terminado con un cuaderno de Jupyter, primero guarde los cambios que desee que permanezcan en el cuaderno. Luego, cierra las ventanas del navegador asociadas con esa sesión de cuaderno de Jupyter. Luego, debe cerrar el servidor local que se abrió para ejecutar el cuaderno Jupyter en la ventana de su terminal. En una Mac o Windows, esto se hace yendo a la ventana de su terminal y escribiendo `Ctrl-C` y luego seleccionando y para sí y presionando Enter.
